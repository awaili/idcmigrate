Network designer (VPC + subnets per account, place servers in VPCs)

Layer 2 · Subnet carving & host placement — runs on top of the active LZ design above: carves each account VPC into tier subnets (web/app/data/infra), sets subnet sizes, groups roles → tiers, and places every server into a subnet + IP. It does not add / rename / merge accounts or change VPC CIDRs — those are Layer 1 (the LZ design above).
carving templates — click to append a starter prompt (stack several), then Generate:
Generate carves the active LZ design's per-archetype VPCs into tier subnets (web/app/data/infra) and places every server into a VPC + subnet + IP. MigraQ proposes the carving policy when you describe it; otherwise the deterministic default is used. Apply makes it the active network design so per-server placement overrides persist.
